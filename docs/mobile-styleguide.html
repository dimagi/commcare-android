When to use each variant
  • Primary — The single main action on a screen (Submit, Save, Confirm). One per screen or card.
  • Secondary — Supporting action alongside a primary (Cancel, Edit). Lower visual weight.
  • Ghost — Optional or navigational actions (Skip, View all, Learn more).
  • Danger — Destructive or irreversible actions only (Delete, Remove). Always confirm before executing.
  • Small — Use inside cards, action bars, or compact layouts where default size is too heavy.

When to use each state
  • Default — Field has not been interacted with
  • Focused — Field is actively selected (on tap)
  • Error — Validation failed. Always pair with an error message below the field. Never rely on color alone.
  • Disabled — Field is not editable. Prefer hiding over disabling when possible.

Usage rules
  • Icons always support a label — do not use as the sole means of communication unless the icon is universally understood (✕ close, ← back)
  • Minimum touch target: 44×44px regardless of icon size
  • Active/selected state: #3B3BCC · Disabled: text-tertiary at 40% opacity
  • Use the existing Figma icon set — do not introduce icons from external libraries without design review
